# FamilyLink

Приложение для улучшения коммуникации между родителями и детьми с помощью AI-помощника.

> ⚠️ **Проект находится в активной разработке.** Название рабочее, может измениться перед публичным релизом.

---

## О проекте

**Проблема.** Родители и дети часто не понимают друг друга: разные поколения, разные интересы, разный опыт. В сложных ситуациях (учёба, гаджеты, эмоции) разговор превращается в конфликт, и обе стороны отдаляются.

**Что делает приложение.** FamilyLink — это **посредник в семейной коммуникации**. Родитель и ребёнок независимо друг от друга описывают ситуацию со своей стороны. AI-помощник, видя обе перспективы, предлагает:
- Как родителю подойти к разговору без давления
- Как ребёнку выразить свои чувства
- Возможные компромиссы
- Материалы (книги, статьи) от проверенных семейных психологов
- Контакты специалистов, если ситуация требует профессиональной помощи

**Что приложение НЕ делает.**
- Не заменяет психолога, врача или педагога
- Не даёт медицинские или психиатрические диагнозы
- Не собирает эмоциональные данные без явного согласия пользователя
- Не передаёт данные третьим лицам без согласия

## Ключевые принципы

### 🔐 Privacy-by-design
Приватность заложена в архитектуре с первого дня:
- Явное согласие на каждый тип обработки данных (отдельное для отслеживания настроения, отдельное для AI-анализа)
- Возможность отозвать согласие в любой момент
- Право на удаление всех своих данных (Right to be forgotten)
- Автоматическая очистка эмоциональных данных старше 6 месяцев
- Аудит согласий с версионированием

### 🛡️ Safety-first для детей
- Регистрация ребёнка только через приглашение родителя
- Автоматическое обнаружение триггеров (насилие, суицидальные мысли, самоповреждение) — при их обнаружении AI не даёт общий совет, а показывает контакты кризисной помощи
- Возможность отметить настроение приватно, без показа семье
- Разделение по ролям (родитель / ребёнок / опекун) с разными правами

### ⚖️ Двусторонняя коммуникация
Приложение — не инструмент контроля родителя над ребёнком, а платформа равного диалога. Обе стороны могут:
- Инициировать обсуждение ситуации
- Отказаться от участия
- Дать/не дать согласие на AI-анализ своего описания
- Удалить свои данные

## Технологический стек

### Backend
- **Java 21** + **Spring Boot 3.5**
- **PostgreSQL 16+** для хранения данных
- **Flyway** для миграций схемы
- **Spring Security + JWT** для аутентификации
- **Spring Data JPA + Hibernate** — ORM
- **Springdoc OpenAPI (Swagger)** для документации API
- **Lombok** для сокращения boilerplate

### AI (в разработке)
- Сейчас — умная заглушка (`MockAiAnalysisService`) с safety-фильтром на триггерных словах и категорийными шаблонами советов
- Планируется — интеграция с внешним LLM (OpenAI / Anthropic / GigaChat / YandexGPT) с обезличиванием данных перед отправкой
- Альтернативно — локальная модель на базе Llama 3.1 / Qwen 2.5 для полной приватности

### Frontend (в планах)
- **React + Capacitor** — кроссплатформенная сборка iOS / Android / PWA из одного кода

## Что уже реализовано

### Аутентификация
- [x] Регистрация с BCrypt-хешированием паролей

- [x] Защита эндпоинтов через JWT-фильтр
- [x] Автоматическое подтягивание пользователя в контроллеры через `@AuthenticationPrincipal`

### Семьи
- [x] Создание семьи с генерацией уникального invite-кода (`FAM-XXXXXX`)
- [x] Вступление в семью по коду с выбором роли
- [x] Роли: PARENT / CHILD / GUARDIAN / OTHER
- [x] Один пользователь может состоять в нескольких семьях (разведённые родители, опекуны)
- [x] Ограничение приватности: только участники семьи видят её данные

### Настроение (Mood)
- [x] Отметка настроения (7 базовых эмоций + опциональная заметка)
- [x] Приватность: настроение можно оставить только себе, не показывая семье
- [x] Просмотр настроений семьи «сегодня» и своей истории
- [x] Автоматическая очистка данных старше 6 месяцев (ежедневная задача в 3:00)
- [x] Обязательное согласие MOOD_TRACKING перед сбором эмоциональных данных

### Ситуации (Situations)
- [x] Создание ситуации с категорией (STUDY, GADGETS, CONFLICTS, EMOTIONS и т.д.)
- [x] Присоединение участников, описания от каждого
- [x] AI-рекомендация на основе описаний с согласием на AI-анализ
- [x] Safety-фильтр: обнаружение триггеров и показ кризисных контактов КР вместо совета
- [x] Проверки для качества совета:
    - Минимум 2 участника с описаниями и согласием на AI
    - Разные пользователи (запрет разговора с самим собой)
    - Разные роли в семье (родитель ↔ ребёнок, не два родителя)
- [x] Кэширование готовой рекомендации (повторный запрос не тратит ресурсы)
- [x] Автосмена статуса ситуации: OPEN → IN_DISCUSSION → RESOLVED

### Согласия (GDPR-compliant)
- [x] Три типа согласий: MOOD_TRACKING, AI_ANALYSIS, DATA_SHARING_WITH_FAMILY
- [x] Явное согласие с версионированием (`v1.0`)
- [x] Возможность отзыва в любой момент
- [x] Аудит: история всех выданных и отозванных согласий
- [x] Right to be forgotten: удаление всех своих эмоциональных данных

### Обработка ошибок
- [x] Централизованный `GlobalExceptionHandler` с единообразными JSON-ответами
- [x] Корректные HTTP-статусы (400/401/403/404/409/500)
- [x] Валидация входных данных через Bean Validation

## Установка и запуск

### 1. Клонировать репозиторий

\`\`\`bash
git clone https://github.com/<ваш-логин>/Family_Linck.git
cd Family_Linck
\`\`\`

### 2. Подготовить базу данных

Подключитесь к PostgreSQL под суперпользователем и выполните:

\`\`\`sql
-- Создать пользователя и базу
CREATE USER familylink WITH PASSWORD 'familylink_dev_password';
CREATE DATABASE familylink OWNER familylink;

-- Дать права на схему public (актуально для Postgres 15+)
\c familylink
GRANT ALL ON SCHEMA public TO familylink;
ALTER SCHEMA public OWNER TO familylink;

-- Создать расширение pgcrypto (нужно право суперпользователя)
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
\`\`\`

### 3. Проверить настройки

Файл `backend/src/main/resources/application.yml` уже настроен для локальной разработки. При необходимости отредактируйте:
- Данные подключения к БД (`spring.datasource`)
- Секретный ключ JWT (`app.jwt.secret`) — **обязательно замените в production**

### 4. Запустить приложение

\`\`\`bash
cd backend
./mvnw spring-boot:run
\`\`\`

Приложение доступно на **http://localhost:8080**

### 5. Открыть Swagger UI

**http://localhost:8080/swagger-ui.html**

Здесь можно протестировать все API-эндпоинты.

## Структура API

Полная документация — в Swagger. Ниже краткий обзор.

### Публичные (не требуют авторизации)
- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход

### Требуют JWT-токен в заголовке `Authorization: Bearer <token>`

**Пользователи**
- `GET /api/users/me` — данные текущего пользователя

**Согласия**
- `POST /api/consents` — дать согласие
- `DELETE /api/consents/{type}` — отозвать
- `GET /api/consents/my` — история моих согласий

**Семьи**
- `POST /api/families` — создать семью
- `POST /api/families/join` — вступить по invite-коду
- `GET /api/families/my` — мои семьи
- `GET /api/families/{familyId}/members` — участники семьи

**Настроение**
- `POST /api/families/{familyId}/mood` — отметить настроение
- `GET /api/families/{familyId}/mood/today` — настроения семьи за сегодня
- `GET /api/families/{familyId}/mood/my` — моя история
- `DELETE /api/mood/my/all` — удалить все мои эмоциональные данные

**Ситуации**
- `POST /api/families/{familyId}/situations` — создать ситуацию
- `GET /api/families/{familyId}/situations` — список ситуаций в семье
- `GET /api/situations/{situationId}` — детали ситуации
- `POST /api/situations/{situationId}/join` — присоединиться
- `POST /api/situations/{situationId}/description` — добавить своё описание
- `POST /api/situations/{situationId}/recommendation` — получить AI-совет

## Разработка

### Обновление схемы БД

Никогда не редактируйте применённые миграции. Для изменений создавайте новый файл:

\`\`\`
backend/src/main/resources/db/migration/V{N}__описание_изменения.sql
\`\`\`

Flyway применит его при следующем запуске.

### Правила именования веток

- `feature/название` — новая функциональность
- `fix/название` — исправление бага
- `refactor/название` — рефакторинг без изменения логики
- `docs/название` — только документация

### Правила коммитов

Формат: `тип(область): краткое описание`

Примеры:
- `feat(mood): add mood tracking with consent system`
- `fix(auth): correct JWT expiration handling`
- `refactor(family): simplify invite code generation`
- `docs(readme): update roadmap`
